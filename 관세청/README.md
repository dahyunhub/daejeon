# 관세청 — 대전 수출입실적 분석

대전광역시의 수출입 구조를 관세청 공공데이터로 정리한 데이터셋. data.go.kr의 **시도별 수출입실적**·**시도별 품목별 수출입실적** API에서 2020~2025년(6개년) 자료를 수집했다.

## 데이터 파일

| 파일 | 단위 | 주요 컬럼 |
|---|---|---|
| `data/daejeon_sido_clean.csv` | 대전 · 연도 | 무역수지, 수출입 건수·금액, 파생지표(억USD, 만건, 건당USD, 증가율) |
| `data/all_sido_by_year.csv` | 전국 17개 시도 · 연도 | 무역수지, 수출입 건수·금액 (원자료) |
| `data/daejeon_share.csv` | 대전 · 연도 | 전국 대비 수입 건수·금액 비중, 전국 순위 |
| `data/daejeon_item_by_year.csv` | 대전 · 연도 × HS품목 | HS 2단위 품목별 수입 건수·금액 |
| `data/daejeon_item_summary.csv` | 대전 · HS품목 | 6개년 합계 + 건수비중 + 건당금액 + 소비재/산업재 구분 |

## 핵심 발견

### 1. 소액·다건 수입으로의 구조 전환

| 지표 | 2020 | 2025 | 변화 |
|---|---|---|---|
| 수입건수 | 71.8만건 | 129.9만건 | +81% |
| 수입액 | 36.2억USD | 41.5억USD | +15% |
| 수입 건당금액 | 5,044 USD | 3,198 USD | **−37%** |
| 전국 수입건수 순위 | 9위 | 10위 | ↓ |

건수는 급증하는 반면 건당 금액은 급락 → 대전이 **개인 소비·해외직구형 수입 도시**로 이동하고 있음을 시사한다. 전국 대비 금액비중은 0.6~0.8%에 머무는데 건수비중(2.29%→2.38%)은 꾸준히 오른다.

### 2. 건수는 소비재, 금액은 산업재

6개년 합계 기준(`daejeon_item_summary.csv`):

- **건수 상위**: HS21 각종 조제 식료품(**건수비중 32.4%**, 건당 113 USD), HS33 화장품, HS19 베이커리, HS62 의류 — 전형적 소액 소비재
- **금액 상위**: HS85 전기기기(건당 5,580 USD), HS84 기계류(건당 11,288 USD) — 대덕연구단지·제조업 설비·부품 조달로 추정되는 고액 산업재

건수와 금액의 주도 품목이 뚜렷이 갈린다.

## 활용 방안

**A. '소비형 수입도시' 진단 리포트**
건당금액 −37% + 건수 +81%를 근거로 대전 무역구조를 재정의. HS21 식품 직구가 흐름을 주도한다는 것을 `daejeon_item_by_year.csv` 시계열로 증명하고, `daejeon_share.csv`의 "건수비중↑·금액비중 정체"로 뒷받침.

**B. 지역산업 모니터링 대리지표**
HS84/85 산업재 수입을 설비·부품 조달 신호로 보고, 연도별 증감을 지역 투자·경기 국면의 프록시로 활용. `all_sido_by_year.csv`로 광주·대구 등 유사 광역시와 벤치마킹.

**C. 부처 융합 분석**
- 관세청 × 우정본부: 식품·잡화 직구 급증 → 국제우편·택배 물량 부하
- 관세청 × 기상청: 계절·이상기후 → 농산물·식품(HS 01~21) 수입 변동
- 관세청 × 코레일: 수입 화물의 지역 물류 이동

**D. 대시보드**
연도 슬라이더 + 품목 트리맵(건수 vs 금액 이중축) + 전국 순위 변화. 건수·금액 괴리를 한 화면에 보여주는 구성.

## 수집 방법

```
python data/customs_collector.py        # 시도별 실적 → daejeon_sido_clean / all_sido_by_year / daejeon_share
python data/customs_item_collector.py   # 품목별 실적 → daejeon_item_by_year / daejeon_item_summary
```

- 엔드포인트
  - 시도별: `https://apis.data.go.kr/1220000/sidotrade/getSidotradeList`
  - 품목별: `https://apis.data.go.kr/1220000/sidoitemtrade/getSidoitemtradeList`
- 연 단위 호출로 1년 제한을 회피하고, `priodTitle='총계'` 행은 제외한 실데이터만 저장한다.
- 실행 전 스크립트의 `SERVICE_KEY`에 data.go.kr 디코딩 키를 넣는다.

> 주의: 현재 수집 스크립트에 API 서비스키가 하드코딩돼 있다. 공개 레포에 올릴 경우 `.env` 등 환경변수로 분리하고 git 히스토리에서 제거할 것.
