# 대전 스마트 드라이포트 물류 데이터 정리

## 프로젝트 주제

해상-내륙 연계형 **대전 스마트 드라이포트**를 기반으로 인바운드 물류 비용을 줄이고, 우정사업본부 대전 IMC의 장비·인력 자원을 선제 최적화하는 AI 플랫폼을 제안한다.

핵심 가설은 다음과 같다.

- 인천항·평택항 등 항만 유입 해외 직구 물량이 도로 직송 중심으로 이동하면서 운송비, 공차, 도착 변동성이 커진다.
- 대전은 전국 내륙 물류의 허브 성격이 강하므로, 항만-철도-드라이포트-IMC를 연결하면 비용과 병목을 줄일 수 있다.
- 우본 물류 데이터는 대전 IMC 자원 최적화와 AS-IS 근거 제시에 직접 활용할 수 있다.

## 바로 쓸 수 있는 근거 포인트

### 1. 대전교환센터는 실제 차량운송 데이터에서 최상위 허브로 나타남

`우편집중국간 차량운송집계데이터_20191031.csv` 기준:

- 발송집중국 출현 빈도 1위: **대전교환센터 4,615건**
- 도착집중국 출현 빈도 1위: **대전교환센터 4,265건**
- 센터 접점 물량 1위: **대전교환센터 약 1,814.8만 건**
- 대전 관련 운송 행: **10,046건**, 대전 관련 총 물량 약 **2,014.7만 건**

제안서에서는 “대전은 내륙 물류 재편의 실증 거점으로 적합하다”는 AS-IS 근거로 활용할 수 있다.

### 2. 2021년 하반기 소포 데이터는 IMC 수요 패턴 분석용으로 큼직하게 쓸 수 있음

`220305 우편물류 접수 및 배달정보(2021년 하반기)` 묶음:

- 등기소포(택배): 약 **1.30억 행**
- 등기소포(창구): 약 **3,969.7만 행**
- 특수통상: 약 **1.14억 행**
- 일반통상: 약 **2,147행**
- 시도별 요약 우편물류정보: 약 **16,934행**
- 전체 CSV 원자료: 약 **2.84억 행**, 약 **15GB**

대용량이므로 분석 시에는 전체 로딩보다 월별·지역별 필터링을 먼저 수행하는 편이 좋다.

추천 필터:

- `접수지역 == 대전광역시`
- `배달지역 == 대전광역시`
- `접수지역 in [인천광역시, 경기도, 충청남도, 충청북도, 세종특별자치시]`
- `배달지역 in [대전광역시, 충청남도, 충청북도, 세종특별자치시]`

### 3. 국제우편 처리물량은 해외직구 인바운드 수요의 보조 지표로 사용

`우체국물류지원단_국제우편 처리물량_20250531.csv`는 2014년부터 2025년 5월까지의 월별 처리물량을 담고 있다. 알리·테무 등 해외 직구 물량 자체를 직접 나타내지는 않지만, 우편 경로로 들어오는 국제 인바운드 물량 증가 추세를 설명하는 프록시로 사용할 수 있다.

## 분석 시 주의사항

- `국내 창구소포 물류이동 OpenAPI`가 승인되면 이 데이터가 본 분석의 1순위가 된다. 현재 CSV 묶음은 보조·사전 탐색용이다.
- 2021년 하반기 시도별 CSV는 파일 수와 용량이 매우 크므로, pandas로 한 번에 전체 로딩하지 말고 지역·월 단위로 나눠 처리해야 한다.
- CSV 인코딩이 `utf-8-sig`와 `cp949` 계열로 섞여 있을 수 있다. 한글이 깨지면 `encoding="utf-8-sig"` 또는 `encoding="cp949"`를 번갈아 확인한다.
- `우편물류 정보(시도별)` 요약 파일은 빠른 탐색에는 좋지만, 전수·건별 분석 근거로 쓰기에는 한계가 있다.
- 국제우편 데이터는 해외직구 전체 물량이 아니라 “우편 경로 국제 처리물량”이므로, 표현할 때는 반드시 프록시라고 명시한다.

## API 승인 후 할 일

1. `우정사업본부_국내 창구소포 물류이동` OpenAPI 호출 스크립트 작성
2. 2024.03~2025.03 기간 중 대전 접수/배달/집배국 관련 데이터 우선 수집
3. 접수시각 기준 시간대별 물량 집계 생성
4. 대전 IMC 도착 예측용 피처 후보 생성
   - 접수일자/시간대
   - 발송·수취 지역
   - 배달 집배국
   - 평일/주말/공휴일
   - 월말·명절·광군제·블랙프라이데이 등 이벤트 플래그
5. 차량운송 데이터와 결합해 AS-IS 도로운송 병목 및 TO-BE 드라이포트 전환 시나리오 작성
