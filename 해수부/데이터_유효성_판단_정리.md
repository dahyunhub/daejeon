## 4. 공통 한계 및 유의사항 (분석 전 필독)

1. **직구 물량 직접 식별 불가** — 4개 데이터 모두 총 물동량/총톤수/선박 단위. 조사시트에서도 해수부 데이터의 '직구 물량 식별'은 전부 **"불가"**로 평가됨. 항만 데이터는 **"인바운드 총량·도착 타이밍의 프록시"**로 사용하고, 직구 순물량은 **관세청 전자상거래 수입통계**(조사시트 ②관세청)로 반드시 보완.
2. **주제의 하류(대전·IMC·철도·기상) 데이터 부재** — TO-BE 아키텍처의 절반(Logistics Routing / IMC Resource Agent) 원천은 이 폴더에 없음. 조사시트의 코레일·우본·기상청 데이터를 별도 확보해야 3-에이전트가 완성됨.
3. **인천항 세분화 비대칭** — 부두/선석별 상세는 **평택당진항만** 존재. 인천항은 NLIC 물동량(항만 총계) 수준까지만 커버.
4. **원천 신고데이터 기간 단기** — PORT-MIS 선박입항신고는 2024.05~2025.07(15개월). ETA 모델 학습엔 충분하나 장기 계절성은 물동량(#1, 77개월)과 결합 권장.
5. **인코딩/명칭 주의** — 선박입항신고 CSV는 **EUC-KR**(UTF-8로 열면 깨짐). 항만명 표기는 `평택.당진`(물동량) / `평택`(신고) / `평택당진`(운영통계)로 소스마다 달라 **조인 시 정규화 필요**.

## 5. 활용 권고

- **즉시 활용 가능(Port Agent 구축)**: `#1 NLIC 물동량`(물량 추세) + `#2 평택당진 운영통계`(부두 캐파) + `#4 선박입항신고`(ETA 원천)의 3종 결합이 이 폴더의 핵심 조합.
- **결합 키**: 항만명(정규화 후) + 연월 기준으로 #1·#3 조인, #4는 건별→월별 집계 후 접합.
- **추가 확보 우선순위**(조사시트 기준): ①관세청 전자상거래 수입동향(직구 직접 식별, 유일) → ②기상청 AWS/해양부이(지연 변수) → ③코레일 철도 O-D·유휴부지(드라이포트 입지) → ④우본 소포 물류이동(IMC 캐파).
